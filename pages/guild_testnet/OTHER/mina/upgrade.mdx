# Update Version

## Berkeley Testnet Release 2.0.0rampup6 (ITN RC2)

<Steps>
  ### Step 1
```bash copy filename="Remove existing Mina sources"
sudo rm /etc/apt/sources.list.d/mina*.list
```

### Step 2 
```bash copy filename="Add and update the Mina package sources"
echo "deb [trusted=yes] http://packages.o1test.net/ focal rampup" | sudo tee /etc/apt/sources.list.d/mina-rampup.list
sudo apt-get update
```

### Step 3
```bash copy filename="Install the Mina Daemon package with the specified version"
sudo apt-get install -y mina-berkeley=2.0.0rampup8-56fa1db
```

### Step 4  

```bash copy filename="Start Mina Daemon"
sudo systemctl start mina
```

### Check and Done

```bash copy filename="Check Mina Status"
mina client status
```

</Steps>

## Exploring Logs for Testworld Mission 2.0 (Testnet Finished)

In this tutorial, we will walk you through the process of exporting and sharing logs for Testworld Mission 2.0 on Mina Protocol Testnet. Follow the steps below to share your logs.

<Steps>
### Step 1: Export Logs

Use the following command to export logs from the Mina client:

```bash copy filename="Export Logs"
mina client export-logs
```

This command will generate log files within the `~/.mina-config/exported_logs` directory.

### Step 2: Upload Logs to Transfer.sh

After exporting the logs, you can share them easily by uploading to Transfer.sh. Use the following command, replacing `#FILE-NAME` with the name of the log file you want to share:

```bash copy filename="Upload Logs"
sudo curl --upload-file ~/.mina-config/exported_logs/#FILE-NAME.tar.gz https://transfer.sh/#FILE-NAME.tar.gz
```

This command will provide you with an output similar to this:

```bash filename="Output"
https://transfer.sh/#RANDOM/#FILE-NAME.tar.gzroot@VPS-NAME
```

</Steps>
